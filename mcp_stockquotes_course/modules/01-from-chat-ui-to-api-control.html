Direct Quote APIs and Setup

The notebook begins like a field technician opening a locked service panel: get credentials, confirm the wiring, then prove the data path works before asking an LLM to narrate anything.

The first architecture choice is to separate access from analysis

This notebook does not start with a finance question. It starts by installing dependencies, loading secrets, and binding a named environment variable for each API key.

Why that matters: if live market access is not stable on its own, every later LLM answer is built on sand. Software engineers care because this is dependency hygiene; product partners care because it explains why “the model failed” is often really “the integration was never ready.”

Plain English

Pull secrets out of Colab’s private notebook storage instead of hardcoding them into visible cells.

Bind each key into a runtime variable so the rest of the notebook can use a stable name.

Choose the model once up front so later calls stay consistent.

Keep the quote provider’s URL separate from the key because one is a service address and the other is a credential.

Create the OpenAI client only after the key exists, which makes failures easier to localize.

Two quote paths teach the same product lesson in different ways

Think of Alpha Vantage as calling the exchange desk directly, while package-based Yahoo Finance is like using a well-trained broker. Both can answer “what is the stock doing,” but one exposes the raw request more clearly and the other packages richer metadata.

Alpha Vantage uses an explicit request with named query parameters and a normalized dictionary return value.

Yahoo Finance wraps retrieval details and returns both price data and business metadata such as market cap and PE ratio.

Code 
def get_stock_quote(symbol: str) -> dict:
    """Fetch the latest stock quote for ``symbol`` using the Alpha Vantage API."""
    params = {"function": "GLOBAL_QUOTE", "symbol": symbol, "apikey": ALPHAVANTAGE_API_KEY}
    response = requests.get(ALPHAVANTAGE_API_URL, params=params, timeout=10)
    response.raise_for_status()
    data = response.json().get("Global Quote", {})
    if not data:
        return {"symbol": symbol.upper(), "price": None}
    return {
        "symbol": data.get("01. symbol", symbol.upper()),
        "price": float(data.get("05. price", "0") or 0.0),
        "change_percent": data.get("10. change percent"),
    }
Plain English

Define one helper whose whole job is “give me the latest quote for this symbol.”

Build a compact request payload with the exact Alpha Vantage function name the provider expects.

Fail fast if the network call itself is broken.

Extract only the `Global Quote` slice instead of making every caller interpret provider-specific JSON.

If no quote comes back, return the same schema anyway so downstream code stays predictable.

Normalize provider field names into business-friendly keys such as `symbol`, `price`, and `change_percent`.

The notebook is teaching contract design, not just ticker lookup

A schema is the quiet hero here. `get_stock_quote()` returns a Python dictionary; `get_stock_quote_yf()` returns JSON text. That difference signals who the expected consumer is and how much cleanup still remains.

Code 
def get_stock_quote_yf(symbol: str) -> str:
    """Get current stock price and basic information."""
    try:
        stock = yf.Ticker(symbol)
        info = stock.info
        hist = stock.history(period="1d")
        if hist.empty:
            return json.dumps({"error": f"Could not retrieve data for {symbol}"})

        current_price = hist['Close'].iloc[-1]
        result = {
            "symbol": symbol,
            "current_price": round(current_price, 2),
            "company_name": info.get('longName', symbol),
            "market_cap": info.get('marketCap', 0),
            "pe_ratio": info.get('trailingPE', 'N/A'),
            "52_week_high": info.get('fiftyTwoWeekHigh', 0),
            "52_week_low": info.get('fiftyTwoWeekLow', 0)
        }
        return json.dumps(result, indent=2)
Plain English

Wrap the provider logic in a `try` block because package-backed integrations can fail in several places.

Ask Yahoo Finance for both identity information and a one-day price history.

Return an explicit error payload if the history table is empty instead of pretending a price exists.

Use the latest closing value as the current price for this notebook’s purposes.

Bundle price, company name, valuation clues, and range data into one result object.

Serialize that object to formatted JSON so it is easy to display or pass along as text.
