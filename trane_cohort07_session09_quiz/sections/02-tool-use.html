From Fixed Workflow to Tool Use

This section covers how the stock-analysis workflow was upgraded from pure prompting to mock and real tool calls.

Study

Tool access increased realism, but orchestration still stayed explicit

After the break, the session returned to the stock-analysis example. The baseline workflow had three specialist agents with different prompts and a fourth synthesis agent that combined their structured outputs into a final report. The next notebooks did not turn that system into a deep agent yet. Instead, they progressively added tool usage.

  • The baseline design sent the same user request to fundamental, technical, and risk agents, each constrained to a fixed schema.
  • Notebook 6 introduced mock tool calls using fake databases to teach the pattern without adding complex external dependencies.
  • Notebook 7 replaced the fake tools with real wrappers such as Yahoo Finance and pandas, while keeping the overall sequence under developer control.
  • Tools often require structured inputs like a ticker symbol, which exposed a usability gap between natural-language user requests and machine-facing tool parameters.
