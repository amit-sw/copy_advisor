Roles, Routing, and Graph Shape

The middle of the deck makes the workflow concrete by naming the roles and showing how the classifier sits at the center of the routing logic.

Study

The architecture becomes visible when the deck names the agents and shows the graph

The example workflow contains five agents: classifier, smalltalk, complaint, status, and feedback. That role list turns the abstract idea of specialization into a concrete topology. The classifier stands at the front of the graph, and the routing logic determines which specialist gets the request before the path ends.

  • The agent names reflect task categories rather than generic assistant personas.
  • The classifier is the entry point that decides the branch.
  • Each specialist path ends after its task is complete in the example graph.
  • The graph is showing workflow structure, not just implementation syntax.
