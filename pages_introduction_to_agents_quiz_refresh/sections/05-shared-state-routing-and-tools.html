Shared State, Routing, and Tools

The final section explains the mechanics that make the workflow executable: a shared state object, explicit routing, and tool usage as part of orchestration.

Study

The agents coordinate through state and graph rules, not by casually chatting with each other

This section answers the practical question left open by the topology slide: how do the agents actually cooperate? The deck's answer is a shared state object with a common function signature, a router that reads fields such as category, and graph edges that make branching explicit. Tool usage fits into the same picture as another conditional behavior the workflow can govern.

  • The communication design is presented in a LangGraph-specific form, but as a generally good pattern.
  • Each agent is modeled as a Python function that accepts the shared state.
  • The example AgentState contains messages, response, and category.
  • The deck contrasts brute-force tool use with more typical conditional tool calling.
