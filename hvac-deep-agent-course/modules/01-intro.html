What This Demo Is Proving

This notebook is a side-by-side lab for three answer styles: plain model output, retrieval-grounded output, and a tool-using agent that adds climate context when a location appears in the query.

Three characters answer the same HVAC question

Think of this file as a live bake-off. One model answers from general knowledge, one must quote retrieved evidence, and one can first ask for climate context before it searches documentation.

Bootstrapping is defensive, not fancy

The notebook first checks normal environment variables, then falls back to Colab secrets. That keeps the demo portable between notebook sessions and regular Python execution.

Code 
def get_secret(name: str, default: str = "") -> str:
    if os.getenv(name):
        return os.getenv(name)
    if userdata is not None:
        try:
            value = userdata.get(name)
            if value:
                return value
        except Exception:
            pass
    return default
Plain English

Create one helper whose whole job is to fetch a named secret.

If the value already exists in the normal process environment, trust that first.

If we are inside Colab, try its secret store next.

If the secret exists there, return it and keep moving.

If Colab secret access fails, do not crash on that step alone.

Fall back to a default value so the rest of setup can decide whether that is acceptable.
