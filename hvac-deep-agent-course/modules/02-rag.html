How RAG Grounds the Answer

Retrieval-augmented generation is the notebook’s evidence lane. It turns a free-form HVAC question into a search against the vector store, then feeds the resulting snippets into a stricter prompt.

RAG is a courier route, not magic

Picture a messenger carrying the user question to a document index, then carrying the best snippets back to the model. The answer becomes more trustworthy because the path is explicit.

The retrieval helper is intentionally literal

Notice what the code does not do. It does not blend hidden heuristics into the answer. It builds search arguments, calls vector search, normalizes the response, and records warnings when evidence is missing.

Code 
kwargs = {
    "vector_store_id": OPENAI_VECTOR_STORE_ID,
    "query": query,
    "max_num_results": int(top_k),
    "rewrite_query": True,
}

try:
    response = openai_client.vector_stores.search(**kwargs)
    results = normalize_search_results(response)
    add_trace(f"Documents found: {len(results)} chunk(s) from vector store {OPENAI_VECTOR_STORE_ID}")
except Exception as exc:
    warning = f"Retrieval failed: {exc}"
Plain English

Build one clean request object for the vector store.

Aim the search at the configured HVAC corpus, not some hardcoded local file.

Pass the user question through directly.

Respect the chosen Top K so the presenter can control result breadth.

Allow query rewriting so search can better match the phrasing stored in the corpus.

Run the search, convert the response into plain Python structures, and log how many chunks came back.

If the retrieval call fails, convert that failure into a visible warning instead of a silent mystery.

🧭
Grounded does not mean perfect

RAG can still retrieve weak evidence. The important improvement is that the weakness becomes inspectable instead of hidden inside pure model confidence.

The RAG prompt narrows the contract

Prompting does not just style the answer. Here it changes the rules: answer only from retrieved evidence, admit incompleteness, and cite filenames in square brackets.

The notebook shows the retrieved rows and the final answer in different sections, which makes debugging much easier.
